Container 101

  • Sanjay Pandey

Sanjay Pandey

14 September 2019 • 1 min read

It is a way to create virtual environment by using an isolated process on host Operating System.

The isolated process has its own set of file system resources and subordinate processes. The container can be thought of as a sandbox to run applications separately without affecting others. In layman terms, it is virtualisation on the top of operating system. A single container might be used to run anything from a small microservice or software process to a larger application. Inside a container are all the necessary executables, binary code, libraries, and configuration files.

Compared to virtualisation they don’t contain the operating system images, so they are light weight and have less overhead.

Why to use ?

Containers have vastly streamlined the application development, build and deployment process. They have less overhead unlike the traditional virtual machines do heavy lifting. They are more efficient as they allow applications to be more rapidly deployed, patched, or scaled. They support agile and DevOps efforts to accelerate development, test, and production cycles. Applications running in containers can be deployed easily to multiple different operating systems and hardware platforms.
